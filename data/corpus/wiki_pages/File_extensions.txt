Minecraft has multiple custom file extensions used to launch the game in specific ways, import packs, or store data.


== Java Edition ==
.mcmeta: Used to define metadata of a resource pack, data pack, or a texture.
.mca: Used to store region data of a world.
.mcr: Previously used to store region data of a world from Beta 1.3 to 1.1.


== Bedrock Edition ==
These file extensions are essentially identical to .zip files and can be converted from or to such a file. However, most of these files allow to launch Minecraft or Minecraft Preview, which will automatically import the file in the dedicated directory.

.mcaddon: Contains other Minecraft packs to import simultaneously as part of an add-on.
.mceditoraddon: Used import an Editor add-on in Bedrock Editor.
.mcfunction: A function containing Minecraft commands. Must be placed inside a behavior pack.
.mcpack: Used to import a resource pack, behavior pack, or skin pack.
.mcperf[more information needed]
.mcproject: Used to import a project in Bedrock Editor.
.mcshortcut: Used to open Minecraft directly in a world or server with the Quick Play feature.
.mcstructure: Used to save a structure with structure blocks. Can be used inside a behavior pack.
.mctemplate: Used import a world template.
.mcworld: Used to import a world.
