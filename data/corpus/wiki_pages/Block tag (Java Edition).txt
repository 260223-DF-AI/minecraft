A block tag is a group of blocks. They are used in advancement and world generation files to determine which block types meet a condition. Some block tags enable special behavior for the contained blocks. They can also be used when testing for block arguments in commands with #<resource location>, which succeeds if the block matches any of the blocks specified in the tag.


== List of tags ==


=== acacia_logs ===
Used in the #logs_that_burn tag.


=== air ===

Special behavior: When placing a hanging sign below blocks in this tag while not sneaking, if the direction matches, the sign is placed with the two parallel chains.

This tag is used in the advancement "Glow and Behold!". Removing signs from this tag cause them to not be able to grant the advancement. === ancient_city_replaceable ===

=== anvil ===
Special behavior:

While using the anvil interface, blocks in this tag have a chance to take damage or break, if they are anvils.
Used in the #mineable/pickaxe tag.

=== mineable/axe ===
Due to usage in the default data component tool, blocks in this tag are mined faster with axes. 
Furthermore, add a block that requires the use of the correct tool (a hardcoded block property) in this tag allow it to be dropped with an axe. However, if the block is also in tags such as #incorrect_for_stone_tool the block will not be dropped by certain axes.


=== mineable/hoe ===
Due to usage in the default data component tool, blocks in this tag are mined faster with hoes. 
Furthermore, add a block that requires the use of the correct tool (a hardcoded block property) in this tag allow it to be dropped with a hoe. However, if the block is also in tags such as #incorrect_for_stone_tool the block will not be dropped by certain hoes.


=== mineable/pickaxe ===
Due to usage in the default data component tool, blocks in this tag are mined faster with pickaxes. Furthermore, if the block requires the use of the correct tool (a hardcoded block property), it needs to be in this tag to be dropped when mined with pickaxes.
Blocks that are also in a tag like #incorrect_for_stone_tool it cannot be dropped by mining it with some pickaxes.


=== mineable/shovel ===
