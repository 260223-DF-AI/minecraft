v0.1.0 alpha is the first publicly released version of Minecraft: Pocket Edition. It was exclusively released for the Sony Xperia PLAY smartphone/gaming console on August 16, 2011. This is the first ever version released in 2011.
This version was planned to release for other Android devices and a prototype build was made with black touch controls and the same menu from v0.1.1 alpha.


== Additions ==


=== General ===
Randomized worlds
Placing and destroying blocks
Ability to invite and play multiplayer on a world (wireless local area network)
Save multiplayer worlds for offline use with the "Copy World" button in the pause menu.
Options for high and low spec phones (Called Fancy Graphics and Lower Graphics Quality, respectively)
Touch Screen controls (officially added in 0.1.1) and auto-jump.

== Demo ==
A demo for Alpha 0.1.0 was released exclusively to the Xperia Play on an unknown date.


== "Canada" Demo ==

On August 17, 2011, an APK labeled mcpe01_canada.apk was uploaded to the pe/ directory of assets.minecraft.net. Despite having the same package name as the full release, it is labeled on the main menu as a demo. Further differences from the demo released on Google Play are unknown, as is the reason for its creation and release.


== Touch Prototype ==
On September 27, 2011, an APK labeled minecraftpe.apk was uploaded to the pe/ directory of assets.minecraft.net. The most notable difference between this build and the Google Play release is that it has a prototype of the touch controls that would later be officially implemented in v0.1.1.
Unlike official releases, the Java code of the APK file is unobfuscated, suggesting that it might be a debug build.


=== Additions ===
Version exclusive: Adds touchscreen controls
New UI for touch screen devices.


== Bugs ==
Fire is invisible, and due to uninitialized memory, can spread to any block, even if not normally flammable, including air.
Stairs are invisible when held
Stairs are invisible when within a 16×16×16 chunk-aligned area with no other blocks
The game will crash when an attempt to render an item in the hotbar is made
TNT will not spawn a primed version of itself in the demo versions (though it tries)
The pictures taken by a camera will include its smoke particles.
